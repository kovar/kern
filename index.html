How to Use

1. Connect to your scale

USB (Chrome/Edge only)
Click USB and select your serial port. Uses the Web Serial API for direct hardware access — no extra software needed. Serial config: 9600 baud, 8N1.
Bridge (any browser)
Start the bridge: uv run bridge.py, then click Bridge. This relays KCP commands over WebSocket so Firefox, Safari, and remote browsers can talk to the scale.

2. Toolbar buttons

Tare (T)
Store the current stable weight as the tare value. Subsequent readings show net weight (gross minus tare). The scale waits for stability before taring.
Zero (Z)
Set a new zero reference point. After zeroing: tare = 0, net weight = gross weight. The scale waits for stability before zeroing.
Read (SI)
Request a single immediate weight reading. Returns the current value regardless of stability — the response indicates "S" (stable) or "D" (dynamic).
Play / Stop
Start or stop automatic polling. Sends SI at the interval shown in the ms field (default 1000 ms). The scale responds with the current weight and unit each time.
Record / Save
Record captures every reading with a timestamp. Save stops and downloads a CSV file.

3. Chart controls

Window sets how much history to show (30 s to 30 min). Y Min/Max lock the vertical axis. Reset returns to auto-scaling. Clear erases all data and resets statistics.

4. Demo mode

Click Demo to generate fake scale data (~250 g with drift and noise) and try out the interface without hardware.

5. Typical workflow

Connect → Tare with empty container → Record → Play → watch live readings and chart → Stop → Save to download CSV.

Running locally

Don't open index.html directly — ES modules need a web server. Run uv run serve.py and open http://localhost:8000.

Custom Command

Send any KCP command. Commands are ASCII, case-sensitive, terminated with CR LF. The scale responds with a status code: A = accepted, L = logical error, I = busy, ES = unknown command.

Response Format

        
Responses follow the pattern: <cmd> <status> <data> CR LF

        

          
            
StatusMeaning

          
          
            
AAccepted / acknowledged

            
LLogical error (invalid parameter)

            
IBusy (currently executing another command)

            
+Overload range

            
-Underload range

            
ESUnknown command or syntax error

          
        

        
Weight responses: S S     100.00 g — status "S" = stable, "D" = dynamic. Value is right-aligned, 10 characters with decimal point.


        
Units

        
g, kg, mg, t (ton), lb, N, kN, tf (ton-force), lbf/klbf, pcs (pieces), %


        
Serial Config

        
9600 baud, 8 data bits, 1 stop bit, no parity. Protocol: KCP v1.1.2.
